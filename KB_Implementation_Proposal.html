Executive Summary

This document proposes the implementation of a Knowledge Base (KB) Service for the Novartis MMF AI project, which enables automated querying of rule books (GPO Rosters, DDD Matrix, ICCR Eligibility Guides) using RAG (Retrieval-Augmented Generation) and LLM technology. The KB service is a critical component that supports Steps 3, 4, and 6 of the 8-step chargeback process automation workflow.

Key Business Value

  • Time Savings: Automates manual rule book lookups, contributing to 10+ minute reduction per chargeback kickout
  • Accuracy: Enables 99.9% automated resolution rate for "Customer not found" kickouts
  • Compliance: Maintains complete audit trail of all KB queries for SOX compliance
  • Scalability: Supports processing 50-200 chargebacks daily with ability to scale

Technical Approach

  • Leverages Existing DocEX Infrastructure: Uses EnhancedRAGService, LLM adapters, and vector databases
  • RAG-Powered Querying: Combines semantic search with LLM generation for intelligent answers
  • Structured Data Extraction: LLM extracts structured data (JSON) for programmatic use
  • Workflow Integration: Seamlessly integrates with chargeback workflow processors

Project Context

Business Context

The KB service directly addresses requirements from the Novartis MMF AI Proposal for automating the 8-step chargeback process:

  • Step 3: Contract Eligibility Validation (queries GPO Roster and Eligibility Guide)
  • Step 4: GPO Roster Validation (queries GPO Roster)
  • Step 6: Class-of-Trade Determination (queries DDD Matrix)

Novartis-Specific Requirements

  • 99.9% vs 0.1% Split: 99.9% of "Customer not found" kickouts result in customer creation (eligible), 0.1% are rejected
  • Priority Order: Highest priority for Federal API calls and GPO Roster pulls
  • Rule Book Updates: GPO Rosters updated on contract/amendment receipt; DDD Matrix updated with contract changes
  • Compliance: Automatic capture of validation results for SOX audit trail
  • Data Lag Handling: Caching and retry logic for federal database API lag issues

Core Components

1. Knowledge Base Service

The core service that wraps RAG capabilities for KB-specific queries. Provides high-level methods for common KB operations.

Key Methods:

  • ingest_rule_book() - Ingest rule books into KB with vector indexing
  • query_rule_book() - Natural language query with RAG-powered search
  • validate_contract_eligibility() - Validate customer contract eligibility (Step 3)
  • get_class_of_trade() - Determine class-of-trade using DDD Matrix (Step 6)
  • get_rule_book_version() - Get version information for rule books

Features:

  • RAG-powered semantic search across rule books
  • LLM-based answer generation with source citations
  • Structured data extraction (JSON) for programmatic use
  • Version control and change tracking
  • Fast lookup by indexed fields

2. Rule Book Processors

Specialized processors for ingesting and processing Novartis rule books from G-Drive/G-Suite.

Components:

  • RuleBookProcessor - Base processor for rule book ingestion
  • GPORosterProcessor - Extract customer contract eligibility from GPO Rosters
  • DDDMatrixProcessor - Extract class-of-trade rules from DDD Matrix
  • EligibilityGuideProcessor - Extract eligibility verification rules

Processing Flow:

  1. Detect rule book type from document name/metadata
  2. Extract text content from document (PDF/Excel)
  3. Use LLM to extract structured data (customers, contracts, COT rules)
  4. Store extracted data as document metadata
  5. Index document in vector database for semantic search

3. Workflow Integration Processors

Processors that integrate KB service into the chargeback workflow.

ContractEligibilityProcessor (Step 3):

  • Extracts contract specifications from chargeback EDI
  • Queries KB service for eligibility validation
  • Stores eligibility result as document metadata
  • Routes to exception queue if not eligible (0.1% case)

COTDeterminationProcessor (Step 6):

  • Uses customer info and federal DB results
  • Queries KB service (DDD Matrix) for COT determination
  • Stores COT code and description for SAP customer creation
  • Flags ambiguous cases for human review

Key Features

RAG-Powered Querying

  • Hybrid Search: Combines vector similarity search with semantic search for better accuracy
  • Context-Aware: LLM generates answers based on retrieved document context
  • Source Citations: Automatic citation of source documents in answers
  • Confidence Scoring: AI-powered confidence assessment for generated answers

Structured Data Extraction

  • JSON Output: LLM extracts structured data in JSON format for programmatic use
  • Type-Specific Extraction: Different extraction schemas for eligibility, COT, contracts
  • Fallback Handling: Text-based parsing if structured extraction fails

Version Control & Change Tracking

  • Rule Book Versioning: Track changes to rule books over time
  • Change Log: Maintain audit trail of rule modifications
  • Rollback Support: Ability to revert to previous rule versions
