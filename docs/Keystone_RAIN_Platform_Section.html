Keystone RAIN Platform Capabilities for MMF Use Cases

Keystone.ai <> Novartis: MMF AI Use Case Scoping Document

The Keystone RAIN platform provides a comprehensive foundation for automating MMF processes through its unified document management, intelligent processing, and workflow automation capabilities. This section outlines how the platform's core capabilities directly address the pain points and solution objectives identified in sections 2.2 and 2.3.

2.6.1: Unified Document Management & Multi-Source Data Ingestion

The platform provides a single, consistent interface for managing unstructured documents from diverse sources, directly addressing the challenge of manual data extraction from PDFs, email attachments, and spreadsheets.

Key Capabilities:

  • Multi-Format Document Support: Native ingestion of PDFs, Excel files, email attachments, and structured data files (NCPDP, EDI) without format-specific preprocessing
  • Flexible Storage Architecture: Support for filesystem, S3, and Google Drive integration, enabling seamless access to GPO Rosters and DDD matrices stored in G-Suite
  • Automatic Document Organization: Basket-based organization allows logical grouping of documents by process stage (e.g., raw_chargebacks, processed_chargebacks, customer_creation_requests)
  • Version Control & Duplicate Detection: Automatic checksum validation and duplicate identification across Model N, SAP, and G-Drive sources, eliminating manual duplicate checks

Application to Use Cases:

  • Chargeback Processing: Ingest chargeback documents from Model N, eligibility guides from G-Drive, and GPO rosters from email attachments into unified platform baskets
  • Medicaid Rebate Processing: Automatically ingest invoices from state web portals, email attachments, and NCPDP flat files into organized baskets by state and quarter
  • Cross-System Integration: Platform acts as central repository, eliminating need for analysts to manually query SAP, Model N, and G-Drive separately

2.6.2: LLM-Powered Structured Data Extraction

Built-in LLM adapters enable automated extraction of structured data from unstructured documents with high accuracy, directly addressing the labor-intensive manual extraction of contract eligibility, customer identifiers, and claim details.

Key Capabilities:

  • Configurable Extraction Prompts: YAML-based prompt templates allow domain experts to define extraction schemas without code changes (e.g., extract contract number, eligibility dates, class-of-trade from GPO rosters)
  • High-Accuracy Extraction: Proven 98%+ accuracy in production invoice processing systems for extracting structured fields (invoice numbers, amounts, dates, line items)
  • Multi-Entity Extraction: Simultaneously extract customer identifiers (HIN, DEA), contract information (Bid Award numbers, eligibility rules), and claim details (NDC, quantities, dates of service)
  • Automatic Metadata Enrichment: Extracted data automatically stored as searchable metadata, enabling downstream queries and validation workflows

Application to Use Cases:

  • Contract Eligibility Extraction: Automatically extract eligibility rules, verification frequency, and class-of-trade requirements from G-Drive eligibility guides
  • Customer Identifier Extraction: Extract HIN, DEA, and address information from chargeback documents and GPO rosters for automated duplicate checking
  • Medicaid Claim Extraction: Extract claim-level details (NDC, quantity, pharmacy NPI, date of service) from state-specific invoice formats, normalizing 30+ different structures automatically
  • External Validation Data Preparation: Structure extracted identifiers for automated queries to DEA, HIBCC, and HRSA federal databases

2.6.3: Entity Matching & Duplicate Detection

Advanced entity matching capabilities automate customer duplication checks across multiple systems, addressing one of the most time-intensive manual steps in the chargeback process.

Key Capabilities:

  • Fuzzy Matching Algorithms: ML-powered entity matching using multiple identifiers (HIN, DEA, address) with configurable similarity thresholds
  • Cross-System Deduplication: Query and match entities across SAP 4 HANA, Model N, and platform storage using extracted identifiers
  • Confidence Scoring: Probabilistic matching scores enable human-in-the-loop review for edge cases while auto-approving high-confidence matches
  • Metadata-Based Matching: Leverage extracted metadata (address, identifiers, class-of-trade) for multi-field matching beyond simple identifier lookup

Application to Use Cases:

  • Customer Duplicate Detection: Automatically check if customer exists in SAP or Model N using extracted HIN/DEA identifiers before creating new customer record
  • GPO Roster Validation: Match customers on GPO rosters against existing SAP customer master data to identify new vs. existing customers
  • Claim Duplicate Detection: Identify duplicate discounts and billing errors in Medicaid claims using NDC, pharmacy NPI, and date-of-service matching

2.6.4: Workflow Automation & Processor Chaining

The platform's extensible processor framework enables complex, multi-step workflows with dynamic logic, supporting the 8-step chargeback process and multi-stage Medicaid rebate workflows.

Key Capabilities:

  • Processor Framework: Extensible processor architecture allows custom processors for each workflow step (e.g., SAP query processor, GPO roster lookup processor, federal database validation processor)
  • Dynamic Processing Logic: Conditional processor execution based on document state (e.g., skip COT determination for existing customers, skip duplicate check if customer already validated)
  • Processor Chaining: Automatic sequencing of processors with dependency management (e.g., extract identifiers → check duplicates → validate eligibility → create customer request)
  • Operation Tracking: Complete audit trail of every processing step with status tracking (pending, in_progress, success, failed) and error logging

Application to Use Cases:

  • Chargeback Workflow Automation: Automate 8-step process: duplicate check → contract eligibility lookup → GPO roster validation → federal database queries → class-of-trade verification → compliance documentation → SAP customer creation → chargeback resolution
  • Medicaid Invoice Processing: Chain processors for invoice ingestion → format normalization → data extraction → duplicate detection → dispute flagging → payment routing
  • Exception Handling: Automatic routing of failed steps to exception queues with full context for analyst review

2.6.5: External System Integration & API Automation

Platform capabilities for external system integration enable automated queries to SAP 4 HANA, federal databases, and third-party APIs, eliminating manual lookups and screenshots.

Key Capabilities:

  • API Integration Framework: Built-in support for REST API calls to external systems (SAP 4 HANA, DEA registrant database, HIBCC HIN portal, HRSA program portal)
  • Credential Management: Secure storage and rotation of API credentials for federal database access (HIN licenses, DEA API keys)
  • Response Processing: Automatic parsing and storage of API responses as document metadata for downstream validation and compliance documentation
  • Retry Logic & Error Handling: Automatic retry for transient API failures with exponential backoff and fallback mechanisms

Application to Use Cases:

  • SAP Customer Queries: Automated queries to SAP 4 HANA to check customer existence using extracted identifiers, with results stored as metadata
  • Federal Database Validation: Automated lookups to DEA, HIBCC (HIN), and HRSA portals to verify class-of-trade, with responses automatically captured for compliance
  • GPO Roster Synchronization: Automated retrieval and processing of GPO rosters from email attachments or portals, with automatic updates to platform storage

2.6.6: Compliance & Audit Trail Management

Built-in operation tracking and metadata management provide complete audit trails required for SOX compliance and government pricing validation, eliminating manual screenshot capture and documentation.

Key Capabilities:

  • Complete Operation History: Every document operation (ingestion, processing, routing, validation) automatically tracked with timestamps, user context, and status
  • Metadata Versioning: Track all changes to extracted data and processing results over time, enabling audit reconstruction
  • Compliance Documentation: Automatic capture of validation results, API responses, and processing decisions as structured metadata
  • Searchable Audit Trail: Query operation history by document, user, date range, or processing stage for compliance reporting

Application to Use Cases:

  • Chargeback Compliance: Automatic documentation of duplicate checks, eligibility validations, and federal database lookups without manual screenshots
  • Medicaid Audit Support: Complete audit trail of claim processing, dispute decisions, and payment routing for state audit requirements
  • SOX Compliance: Full traceability of customer creation requests from chargeback kickout through SAP customer creation with all validation steps documented

2.6.7: Scalable Architecture for Multiple Use Cases

The platform's reusable components and extensible architecture enable scaling from chargeback automation to Medicaid rebate processing and future MMF use cases without rebuilding core infrastructure.

Key Capabilities:

  • Reusable Processors: Processors developed for one use case (e.g., identifier extraction) can be reused across multiple workflows (chargeback, rebate, forecasting)
  • Basket-Based Organization: Logical document grouping enables parallel processing of multiple use cases without data mixing
  • Metadata Schema Flexibility: Key-value metadata system accommodates different data structures across use cases without schema changes
  • Horizontal Scalability: Platform architecture supports processing 50-200 chargebacks daily with ability to scale to thousands of Medicaid claims per quarter

Application to Use Cases:

  • Chargeback to Rebate Extension: Processors for data extraction, entity matching, and external validation developed for chargebacks can be adapted for Medicaid claim processing
  • Forecasting Data Preparation: Extracted and encoded data from chargeback and rebate workflows can feed into forecasting models without additional data preparation
  • Future Use Case Expansion: Platform infrastructure supports additional MMF use cases (government pricing, GTN forecasting) with minimal new development

2.6.8: Integration with Keystone RAIN Data Encoding

The platform seamlessly integrates with Keystone's RAIN encoding system, ensuring extracted and processed data is encoded in reusable features for downstream analytics and forecasting.

Key Capabilities:

  • Automatic Feature Encoding: Extracted data (class-of-trade, contract ID, eligibility dates) automatically encoded into RAIN feature format
  • Cross-Use Case Reusability: Encoded features from chargeback processing (customer class-of-trade, contract eligibility) reusable for rebate processing and GTN forecasting
  • Temporal Feature Tracking: Eligibility dates and contract periods encoded with temporal semantics for time-series forecasting
  • Hierarchical Feature Support: Features encoded at multiple grains (customer, contract, program) enabling aggregation for different forecasting targets

Application to Use Cases:

  • Chargeback to Forecasting Pipeline: Customer class-of-trade and contract eligibility encoded during chargeback processing automatically available for rebate and GTN forecasting models
  • Medicaid Claim Feature Engineering: Claim-level features (NDC, pharmacy, program) encoded for both dispute prediction and utilization forecasting
  • Unified Data Model: Single encoding schema across chargeback, rebate, and forecasting use cases eliminates duplicative data preparation
